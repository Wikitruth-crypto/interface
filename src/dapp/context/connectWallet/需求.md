# WikiTruth Connect Wallet UI 需求说明

本文档定义基于纯 CSS/HTML 的连接钱包组件需求，整体视觉参考 Reown AppKit 暗色系风格，但需脱离 Reown 依赖，便于在其他项目中复用。

## 1. 交互流程概览

1. “Connect” 主按钮：常驻在导航或 DApp 入口区，展示连接状态（未连接/已连接地址/加载中）。
2. 点击后弹出 `Connect Modal`：
   - 顶部标题、关闭按钮、辅助提示（“支持 Sapphire 网络钱包”）。
   - 默认页签为“选择钱包”：展示主流钱包列表 + 关键状态（已安装/支持多链/二维码）。
   - 搜索输入框可过滤钱包名称，底部显示 “+510 wallets” 类似提示，链接到“更多钱包”说明文档。
3. 已连接后，主按钮切换为“账户摘要”：点击弹出 `Account Modal`，包括资产概览、快捷操作、断开连接。
4. `Network Modal`：从 `Account Modal` 切换网络入口进入，展示 DApp 支持的链（Sapphire Testnet/Mainnet 置顶）。

## 2. 组件拆分

1. `ConnectButton`
   - 状态：`idle`（Connect Wallet CTA）、`connecting`（loading spinner + “Connecting…” 文案）、`connected`（缩写地址 + 链徽章），`error`（失败提示、重试按钮）。
   - 展示当前链 Icon（来自链配置）、余额简写（可选）。
2. `ConnectModal`
   - 头部：标题、帮助 icon（打开文档）、关闭按钮。
   - `WalletList`：每行包含图标、名称、副标签（`INSTALLED`/`MULTICHAIN`/`QR CODE`）、悬浮态背景。列表需要支持滚动。
   - `SearchWalletInput`：带放大镜 icon，占位文本“Search wallet (500+)”。
   - `Secondary Actions`（可选）：例如“WalletConnect QR” 或 “输入自定义 RPC”。
   - 禁用 Email/Google 登录（明确写在文案：“暂不支持邮箱社交登录”）。
3. `AccountModal`
   - 头部：链图标 + 下拉（指向 Network Modal）、关闭按钮。
   - Avatar：圆形渐变背景（可根据地址 hash 生成颜色）。
   - 地址行：显示缩写地址 + 复制按钮 + “查看区块浏览器”链接。
   - 余额：主资产数值 + 单位（ROSE/ETH）。
   - 快捷操作：`Fund`, `Swap`, `Send`, `Activity`、`Disconnect`（红色 icon），每行采用 icon + 文案 + chevron。
4. `NetworkModal`
   - 顶部搜索框（“Search network”）。
   - `NetworkList`：链图标、名称、副标题（`Default`、`Popular` 等），当前链右侧标记 `Connected`。
   - 底部警示文案：“钱包可能不支持所有网络，请在钱包内确认”。

## 3. 状态与逻辑需求

| 场景 | 反馈要求 |
| --- | --- |
| 连接中 | Modal 禁用关闭按钮 1 秒，显示进度行 & `Connecting to MetaMask…` |
| 用户拒绝 | Banner 提示“已取消连接”，保持 Modal 打开 |
| 连接失败 | Banner 提示 + 重试按钮，自动收集错误栈（便于日志） |
| 钱包未安装 | `INSTALLED` 标签变灰，点击跳转安装指南（新开 Tab） |
| 多链钱包 | `MULTICHAIN` 标签（灰色），提示可在钱包内切换链 |
| QR 模式 | 若钱包支持二维码，副按钮显示 `QR CODE`，点击展开二维码面板 |
| 断开连接 | 需要二次确认弹窗：“确定要断开 0x57…862B 吗？” |

## 4. 样式规范

| 元素 | 风格 |
| --- | --- |
| 基础色 | 背景 #121212，卡片 #1E1E1E，Edge #2C2C2C，字体 #F5F5F5 |
| 圆角 | 按钮与卡片 16px，输入框 12px |
| 阴影 | `0 20px 60px rgba(0,0,0,0.45)` |
| 字体 | Inter / SF Pro，`Connect` CTA 使用 600 字重 |
| 列表行 | 高度 60px，左侧 Icon 40x40（圆角 12px），右侧标签 10px 字号 |
| 交互 | 悬浮高亮（背景 #2A2A2A），点击波纹（使用 CSS 伪类实现） |
| 动画 | Modal 弹出使用 scale+fade 200ms，列表滚动自定义滚动条 |

## 5. 可复用 CSS 结构

建议拆分以下 BEM 命名空间，便于在其他项目中复用：

- `.wt-connect-button`
- `.wt-modal` / `.wt-modal__header` / `__body` / `__footer`
- `.wt-wallet-item`
- `.wt-network-item`
- `.wt-status-badge`（区分 success / warning / info）
- `.wt-avatar`（根据地址 hash 赋色）

## 6. 文案与可访问性

- 所有按钮提供 `aria-label`，输入框关联 `aria-describedby`。
- Modal 需要 trap focus，支持键盘 Esc 关闭。
- 错误提示需提供“复制错误详情”按钮，方便支持反馈。
- 多语言：暂不实现国际化，但文案存放在常量表，后续可接入 i18n。

## 7. 开发交付

1. 先实现静态版（无逻辑）以锁定视觉。
2. 整合 wagmi hooks：`useConnect`, `useDisconnect`, `useAccount`, `useSwitchChain`。需封装一个“连接逻辑适配层”（如 `ConnectController`），内部依赖当前的 `@oasisprotocol/sapphire-wagmi-v2`；对外暴露统一接口，方便未来切换或并存其它 SDK（例如 thirdweb、RainbowKit、自研 RPC 客户端）时只替换适配实现。
3. 设计 token：颜色/尺寸/阴影变量写在 `connect-wallet.css`，避免散落。
4. 编写 Storybook/示例页面，覆盖以下状态：`未连接`、`连接中`、`已连接`、`切换网络`、`错误`。

以上需求文档用于指导 UI 设计与前端实现，确保替换 Reown 组件后仍保持一致体验并支持 Oasis Sapphire 加密链路。

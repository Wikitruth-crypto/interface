## 1. 模块概述
WikiTruth DApp 的核心页面之一，位于 `/app/marketplace` 路由，用于展示和浏览链上的 TruthBox 列表。


## 2. 核心需求描述

### 2.1 核心功能（按优先级排序）

#### P0 - 统计数据展示
- 页面顶部显示全局统计数据卡片
- 展示指标：Supply（总供应量）、Storing（存储中）、OnSale（在售）、Swaping（交换中）、InSecrecy（保密中）、Published（已发布）、Volume（总交易量）
- 数据从 Supabase `statistical_state` 表获取
- 加载时显示默认值 0，加载完成后更新为实际数据
- 支持响应式布局，移动端自动换行

#### P0 - TruthBox 列表展示
- 以网格形式展示 TruthBox 卡片
- 响应式布局：
  - 移动端：1 列
  - 平板（sm）：2 列
  - 桌面（lg）：3 列
  - 大屏（xl）：4 列
- 每个卡片显示：标题、图片轮播（boxImage + nftImage）、国家/州、事件日期、Token ID、价格（如适用）、状态标签
- 点击卡片跳转到详情页 `/app/boxDetail/{id}`
- 卡片使用渐进式加载动画（fade-scale），等待图片加载完成后才显示

#### P0 - 数据获取与缓存
- 从 Supabase 数据库的 `search_boxes` RPC 函数获取数据
- 使用 React Query 进行数据获取和缓存
- 默认排序：无搜索时按 `box_id` 降序（最新在前），有搜索时按 `relevance` 降序
- 支持筛选条件变化时自动重新查询
- 查询结果缓存 5分钟（`src\config\queryClient.ts`）

#### P0 - 筛选功能
- **搜索框**：支持关键词搜索（当前禁用状态，预留接口）
- **状态筛选**：下拉选择框，选项包括 All、Storing、Selling、Auctioning、Paid、Refunding、InSecrecy、Published
- **排序方式**：下拉选择框，选项包括 Default、Price ↑、Price ↓、Date ↑、Date ↓、ID ↑、ID ↓
- **价格范围**：双输入框，支持设置最小值和最大值（支持 3 位小数）
- **ID 范围**：双输入框，支持设置起始 ID 和结束 ID（整数）
- **日期范围**：日期选择器，支持选择开始日期和结束日期
- **国家/州选择**：级联选择器，先选择国家，再选择州
- 筛选条件变化时自动触发数据重新查询
- 显示当前筛选结果的总数

#### P1 - 分页功能（双模式）
- **分页器模式（Paginator）**：
  - 使用 Ant Design Pagination 组件
  - 支持切换页码和每页数量
  - 显示总数据量和当前页码信息
- **无限滚动模式（Load More）**：
  - 滚动到底部时自动加载更多数据
  - 触发距离：距离底部 100px 时触发
  - 每次加载 4 条数据（loadBatchSize）
  - 加载中显示 Loader 组件
  - 加载失败显示 Retry 按钮
  - 没有更多数据时显示 "No More" 提示
- 两种模式通过 Zustand store 管理，可切换（当前 UI 中分页模式选择器被注释）

#### P1 - 渐进式加载动画
- 卡片使用骨架屏（SkeletonCard）作为占位符
- 等待图片加载完成后才显示真实内容
- 使用动画效果，过渡时间 300ms
- 避免重复触发动画（通过 itemsKey 判断数据是否真正变化）
- 图片加载完成后通知父组件，触发下一张卡片的显示

#### P1 - 空状态和加载状态
- 初始加载时显示 "Loading..." 提示
- 无数据时显示 "No boxes found" 提示
- 加载状态通过 React Query 的 `isLoading` 和 `isFetching` 管理

### 2.2 技术约束

**数据存储**：
- 筛选条件和分页配置存储在 Zustand store 中（`marketplaceStore`）
- 数据从 Supabase 数据库实时查询，不进行本地持久化
- 使用 React Query 缓存查询结果，减少重复请求

**数据源**：
- Supabase RPC 函数：`search_boxes`（查询列表）、`statistical_state` 表（统计数据）

**兼容性**：
- 支持现代浏览器（Chrome、Edge、Firefox、Safari 最新版本）
- 移动端支持 iOS Safari 14+、Chrome Mobile 90+
- 响应式设计，适配各种屏幕尺寸

**性能要求**：
- 列表渲染 100+ 条数据时无明显卡顿
- 图片使用 IPFS 网关加载，支持渐进式加载
- 使用 React Query 缓存，减少重复请求
- 分页加载，避免一次性加载过多数据

### 3.2 体验验证

**响应式适配**：
- ✅ 移动端（< 640px）：单列布局，筛选区域垂直排列
- ✅ 平板（640px - 1024px）：2-3 列布局，筛选区域部分横向排列
- ✅ 桌面（> 1024px）：3-4 列布局，筛选区域完全横向排列
- ✅ 所有屏幕尺寸下，卡片不溢出，文字不换行异常

**加载体验**：
- ✅ 初始加载显示 "Loading..." 提示
- ✅ 卡片使用骨架屏占位，避免布局跳动
- ✅ 渐进式加载动画流畅，无卡顿
- ✅ 图片加载失败时，卡片仍正常显示（不阻塞）

**交互反馈**：
- ✅ 卡片 hover 时显示边框高亮
- ✅ 筛选条件变化时，列表平滑更新（使用 keepPreviousData）
- ✅ 分页切换时，页面滚动到顶部（可选）
- ✅ 所有可交互元素有明确的视觉反馈

### 3.3 边界情况验证

**数据边界**：
- ✅ 列表为空时，显示空状态提示
- ✅ 列表数据量很大（1000+）时，分页正常工作
- ✅ 筛选结果为空时，显示 "No boxes found"
- ✅ 价格/ID 范围无效时，正确处理（如 min > max）

**异常情况**：
- ✅ Supabase 连接失败时，显示错误提示（控制台）
- ✅ 网络请求超时时，允许重试
- ✅ 图片加载失败时，卡片仍正常显示
- ✅ 筛选条件包含特殊字符时，正常处理

**性能边界**：
- ✅ 快速切换筛选条件时，不触发重复请求（防抖）
- ✅ 列表渲染 100+ 条数据时，滚动流畅
- ✅ 图片加载不阻塞页面渲染（渐进式加载）
